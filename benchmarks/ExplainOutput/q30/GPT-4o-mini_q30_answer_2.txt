The last expression will return `[11.0; 21.0; 6.0]`.

Explanation: 
1. Initially, `sales_tax` is `1.0`.
2. When `update_sales_tax ()` is called, it returns `2.0`, but it does not change the global `sales_tax` variable because in OCaml, top-level values are immutable.
3. The `update_price_list price_list` function calls `List.map add_tax price_list`.
4. The `add_tax` function still refers to the original `sales_tax` value, which is still `1.0`.
5. Thus, the computed prices become:
   - For `10.0`: `1.0 +. 10.0 = 11.0`
   - For `20.0`: `1.0 +. 20.0 = 21.0`
   - For `5.0`: `1.0 +. 5.0 = 6.0`
6. Therefore, the final result of `update_price_list price_list` is `[11.0; 21.0; 6.0]`.
