**Solution:**

`[11.; 21.; 6.]`

**Explanation:**

1.  `update_sales_tax ()` calculates `sales_tax *. 2.0` (which is `1.0 *. 2.0 = 2.0`). However, it **does not** update the global `sales_tax` variable because OCaml variables are immutable after initialization unless explicitly made mutable via `ref`.

2.  `update_price_list price_list` then uses the original `sales_tax` which is `1.0` in a `map` function.  So the function will add 1.0 to each item in `price_list`.
    *   10.0 + 1.0 = 11.0
    *   20.0 + 1.0 = 21.0
    *   5.0 + 1.0 = 6.0

Therefore, the resulting list is `[11.; 21.; 6.]`.
